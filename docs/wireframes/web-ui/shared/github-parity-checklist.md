# GitHub Parity Checklist

Use this checklist for every wireframe page.

## Controls

- [ ] Primary and secondary button hierarchy matches GitHub patterns.
- [ ] Danger actions are visually distinct and require confirmation.
- [ ] Focus-visible style is clearly present.
- [ ] Disabled and busy states are represented.

## Lists and Tables

- [ ] Filter/search controls are at the top of list panels.
- [ ] Row actions are contextual and not hidden behind ambiguous placement.
- [ ] Status badges are consistent and readable.

## Forms

- [ ] Labels, helper text, and error states follow GitHub rhythm.
- [ ] Save/cancel/delete actions are consistently positioned.

## Editors

- [ ] Multi-line editing uses Monaco pattern, not textarea fallback.
- [ ] Editor toolbar/status line are represented.
- [ ] Keyboard-driven usage is documented.

## Decision Rule

- [ ] If unclear, choose the GitHub-like option and record the choice.
